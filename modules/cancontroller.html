CANController¶

API¶

class modules.cancontroller.CANController(ecus: dict, can_spec_path: str, channel: str, bitrate: int = 500000)¶

High level python object to interface with hardware.

The CANController works passively for reading signals; you create one with a dictionary of ECUs, and it will automatically keep all of their CAN states updated according to the CAN traffic it reads.

Confluence

Parameters

  • ecus (dict) – A dictionary of str : ECU pairs. The CANController needs this to automatically update the states of those ECUs. The keys in this dictionary should match the names of the senders in your can spec file.

  • can_spec_path (str) – The path to the can spec. For now, we only support .dbc files. Should be stored in artifacts.

  • channel (str) – The name of the can channel to use. If using a virtual can bus, use vcan0 or similar; if using a real CAN bus, use can0 or similar. You can use $ ip link show to view options.

  • bitrate (int) – The bitrate of the can bus. Defaults to 500K If using a virtual bus, you can ignore this.
